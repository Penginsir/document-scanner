# ドキュメントスキャナー

スマホのブラウザでカメラを起動し、書類を撮影してテキスト層付きPDFを1つ出力するWebアプリです。ビルド不要の静的サイト(HTML/CSS/JS)で、サーバー処理は一切ありません(OCR・PDF生成はすべて端末内で完結します)。

## 仕組み

1. カメラで書類を1枚撮影
2. Tesseract.js(日本語+英語)で端末内OCRを実行
3. 単語ごとの認識信頼度(confidence)から「認識失敗率」を計算
   - 信頼度が60未満の単語を「失敗」とみなし、文字数ベースで失敗率を算出
   - **失敗率が70%を超えたらスキャンを拒否**(通常の品質ゲート。読み取れない書類でPDFを作らせない)
   - 70%以下ならOK → 撮影画像を背景にした、検索・コピー可能な透明テキスト層付きPDFを生成してダウンロード

閾値は [app.js](app.js) 冒頭の `WORD_CONFIDENCE_THRESHOLD` / `MAX_FAILURE_RATE` で調整できます。

## ローカルで動かす

カメラ(`getUserMedia`)は **HTTPS または localhost** でないと動作しません。単純に `index.html` を `file://` で開いても動きません。

```powershell
# このフォルダで実行(Python同梱環境の場合)
python -m http.server 8000
# または Node.js がある場合
npx http-server -p 8000
```

PC のブラウザで `http://localhost:8000` を開けば動作確認できます。

## スマホの実機で試す

スマホの実ブラウザでカメラを使うには、スマホから見て HTTPS でアクセスできる必要があります。手軽な方法:

- **ngrok / Cloudflare Tunnel** などでローカルサーバーをHTTPS公開する
- **GitHub Pages / Netlify / Vercel** などにこのフォルダをそのままデプロイする(すべて静的ファイルなのでNode不要)
- PCとスマホが同じネットワークにあり、Chromeの `chrome://flags` で該当オリジンを安全なコンテキスト扱いにする(開発時のみ推奨)

## ファイル構成

- `index.html` — 画面構造(スタート→カメラ→プレビュー→処理中→拒否/成功)
- `style.css` — スタイル
- `app.js` — カメラ制御・OCR・PDF生成ロジック
- `fonts/NotoSansJP-Regular.ttf` — PDFの透明テキスト層に埋め込む日本語フォント(OFLライセンス。`pdf-lib`のサブセット機能で実際に使われたグリフのみが最終PDFに埋め込まれます)

外部ライブラリ(`tesseract.js` / `pdf-lib` / `@pdf-lib/fontkit`)はjsDelivr CDN経由で読み込みます。画像データそのものは外部送信されず、OCR・PDF生成は端末内(ブラウザ)で完結します。

## 既知の制約

- 1回のスキャンにつき1ページ・1PDFのみ(複数ページ結合には非対応)
- 認識失敗率は単語単位の信頼度ベースの推定値であり、目視の読みやすさと完全には一致しません
- 初回アクセス時にOCR言語データ(jpn+eng)とNoto Sans JPフォント(約9MB)をダウンロードするため、Wi-Fi環境での初回利用を推奨します(以降はブラウザキャッシュが効きます)
